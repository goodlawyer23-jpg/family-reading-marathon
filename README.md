# 우리가족 독서마라톤

우리가족 독서마라톤은 가족 러너가 각자 읽는 책과 페이지 수, 감상, 좋았던 문장을 기록하고, 가족 전체의 독서 목표 달성 과정을 마라톤처럼 시각화하는 Streamlit 웹페이지입니다.

1차 MVP는 거대한 상용 서비스가 아니라, 개인 컴퓨터에서 실행하고 경진대회 발표에서 안정적으로 시연할 수 있는 작은 웹서비스를 목표로 합니다.

## 주요 기능

- 러너 등록, 수정, 삭제
- CSV 자동 생성 및 CSV 기반 저장
- 샘플 데이터 생성 버튼
- 네이버 책 검색 API 검색
- 검색 화면과 직접등록 화면 분리 표시
- 검색 결과는 기본 최대 50개로 검색
- 검색 결과는 한 화면에 10개씩 페이지 번호 버튼으로 구분 표시
- 책 검색 결과의 긴 책 소개는 기본 접힘 상태로 표시
- 페이지 번호 버튼 클릭 시 검색 결과 선택 구역으로 자동 스크롤
- 책 제목 검색 / ISBN 검색 선택
- API 실패 시 샘플 책 검색 자동 전환
- 책 직접 등록
- 책 등록 시 읽을 러너 연결
- 책장에 추가하기 전 검색 결과 카드에서 전체 페이지 수 직접 입력
- 가족 책장 카드 보기
- 가족 책장에서 전체 페이지 수 수정
- 오늘의 독서 기록 통합 입력
- 선택한 러너의 책장에 등록된 책만 기록 가능
- 시작/끝 페이지 기준 읽은 페이지 수 자동 계산
- 러너+책별 마지막 끝 페이지 기반 시작 페이지 자동 제안
- 페이지 수 직접 입력 방식 지원
- 좋았던 문장 선택 입력, 완독 시 별점과 한 줄 감상 입력
- 기록 모아보기에서 독서 기록, 좋았던 문장, 한 줄 감상 조회
- 독서 기록 2단계 확인 후 삭제
- 가중치 반영 페이지 자동 계산
- 홈 대시보드 가족 전체 마라톤 트랙 표시
- 우리 가족 기여도 카드 표시
- 러너 카드와 규칙 기반 협동 배지 표시
- 가족 전체 목표 대비 진행률 표시
- 자세히 보기에서 러너별 누적 페이지와 기여 현황 표시
- 규칙 기반 월간 독서 리포트 생성
- 국립중앙도서관 ISBN API는 선택 기능으로만 수동 테스트

## 샘플 데이터 사용 방법

앱을 처음 실행한 뒤 홈 화면의 **개발/시연용 도구** expander를 열고 **샘플 데이터 생성 / 초기화** 버튼을 누르면 다음 데이터가 생성됩니다.

- 러너 4명
- 샘플 도서 7권
- 최근 7일 독서 기록
- 좋았던 문장 3개
- 한 줄 감상 3개
- 2026년 7월 독서마라톤 목표 2,000페이지

이 버튼만으로 API 키 없이도 전체 화면 시연이 가능합니다.


## 홈 대시보드 UI

홈 화면은 “가족이 함께 독서마라톤 코스를 달리는 느낌”을 주도록 구성되어 있습니다.

- 가족 전체 누적 페이지와 목표 페이지를 기준으로 큰 마라톤 트랙을 표시합니다.
- 가족 전체 트랙의 러너 이모지는 오른쪽으로 달리는 `🏃‍➡️`를 사용합니다.
- 개별 경쟁 트랙 대신 **함께 만든 독서 거리** 섹션에서 러너별 협동 기여도를 퍼센트 카드로 표시합니다.
- 기여도는 `러너 weighted_pages ÷ 가족 전체 weighted_pages × 100`으로 계산합니다.
- 러너 카드에는 실제 읽은 페이지, 기여 반영 거리, 기록일 수, 규칙 기반 협동 배지가 표시됩니다.
- 샘플 데이터 생성 버튼은 홈 화면의 **개발/시연용 도구** expander 안에 있습니다.
- 배지는 기존 `reading_logs.csv`, `reviews.csv`, `quotes.csv`만 사용해 계산합니다.
- 샘플 데이터 생성 / 초기화 버튼은 실제 사용 화면을 방해하지 않도록 **개발/시연용 도구** expander 안에 있습니다.

## 러너 관리

**설정 / 러너 관리** 화면에서 새 러너를 추가하거나 기존 러너 정보를 수정·삭제할 수 있습니다. 수정 가능 항목은 이름, 이모지/avatar, 역할, 러너 유형, 가중치입니다. 이모지는 기본 후보 목록에서 드롭다운으로 선택합니다. 러너 유형은 성인 / 청소년 / 어린이 / 유아 4개만 사용합니다. 수정한 정보는 `family_members.csv`에 저장되며 홈 대시보드, 가족 책장, 오늘의 독서 기록 화면에 반영됩니다.

러너 삭제는 실수 방지를 위해 2단계 확인 방식으로 처리합니다. 삭제를 확정하면 해당 러너의 책장, 독서 기록, 좋았던 문장, 완독 감상도 함께 정리되어 대시보드와 월간 리포트 계산에서 제외됩니다. 테스트용으로 추가한 러너를 정리할 때 사용할 수 있습니다.

### 러너 유형과 기본 가중치

기본 러너 유형과 권장 가중치는 다음 기준으로 설정했습니다. 값은 기본 제안일 뿐이며, 러너 추가/수정 화면에서 직접 조정할 수 있습니다.

| 러너 유형 | 기본 가중치 | 기준 설명 |
|---|---:|---|
| 성인 | 1.0 | 성인 책 한 권을 약 300쪽 기준으로 봅니다. |
| 청소년 | 1.1 | 성인과 비슷하지만 독서 난이도 차이를 조금 반영합니다. |
| 어린이 | 1.4 | 『마틸다』 같은 어린이 장편 250쪽을 읽으면 약 350쪽으로 반영됩니다. |
| 유아 | 2.0 | 짧은 그림책도 가족 독서마라톤에 의미 있게 반영합니다. |

가족 월간 목표의 기본값은 2,000쪽입니다. 성인은 월 2권을 기본 페이스로, 월 4권은 주 1권 챌린지로 볼 수 있습니다.

## API 키 설정 방법

1차 MVP는 API 키가 없어도 작동합니다.

네이버 책 검색 API를 연결하려면 환경변수 또는 `.env` 파일에 아래 값을 설정합니다. 앱 실행 시 `python-dotenv`가 `.env`를 읽고, 값이 있으면 책 검색 화면에서 네이버 API를 먼저 사용합니다.

국립중앙도서관 ISBN 서지정보 API는 선택 기능입니다. 일부 PC 또는 네트워크 환경에서는 연결이 지연되거나 실패할 수 있으므로, 책장 추가 시 자동 호출하지 않습니다. 필요할 때만 설정 화면 또는 `test_nlk_api.py`에서 수동으로 테스트합니다.

```bash
NAVER_CLIENT_ID=발급받은_CLIENT_ID
NAVER_CLIENT_SECRET=발급받은_CLIENT_SECRET
NLK_CERT_KEY=발급받은_국립중앙도서관_인증키
```

`.env` 파일은 GitHub에 올리지 않도록 `.gitignore`에 포함되어 있습니다.

Streamlit Community Cloud에 배포할 때는 앱 설정의 Secrets에 아래처럼 입력합니다.

```toml
NAVER_CLIENT_ID = "발급받은_CLIENT_ID"
NAVER_CLIENT_SECRET = "발급받은_CLIENT_SECRET"
NLK_CERT_KEY = "발급받은_국립중앙도서관_인증키"
```

현재 `app.py`는 로컬 `.env` / 환경변수와 Streamlit Secrets를 모두 확인합니다. 네이버 API 키가 없거나 API 호출에 실패하면 앱이 멈추지 않고 샘플 책 검색으로 자동 전환됩니다. 국립중앙도서관 API는 기본 자동 조회가 꺼져 있습니다. 따라서 국립중앙도서관 API 실패가 책 등록, 책장 추가, 가족 책장 반영을 방해하지 않습니다. 책장 추가 시 `total_pages`는 네이버/샘플/직접 입력값을 사용하고, 값이 없으면 0으로 저장합니다.

책 검색 화면에서는 검색 결과 개수를 따로 고르지 않아도 됩니다. 앱은 기본적으로 네이버 API의 `display=50`을 사용해 최대 50개를 검색하고, 샘플 검색 fallback도 최대 50개 기준으로 작동합니다. 검색 방식은 **책 제목 검색**과 **ISBN 검색** 중 선택할 수 있으며, ISBN 검색은 하이픈이 포함된 값도 처리합니다. 검색어 입력창에서 Enter를 눌러도 검색이 실행되고, 기존 **검색하기** 버튼도 그대로 사용할 수 있습니다. 검색어가 비어 있으면 검색하지 않고 안내 메시지를 표시합니다. 검색 결과가 10건을 초과하면 화면에는 10개씩 나누어 표시되고, 책 목록 아래의 `1 2 3 4 5` 형태 페이지 번호 버튼으로 다음 결과를 볼 수 있습니다. 페이지 번호를 누르면 다음 결과를 바로 확인할 수 있도록 `3단계. 원하는 책을 선택해 책장에 추가하세요` 위치로 자동 스크롤됩니다. 각 검색 결과 카드에서는 표지, 제목, 저자, 출판사, ISBN, 출간일, 전체 페이지 입력칸, 책장 추가 버튼만 바로 보이고, 긴 책 소개는 `책 소개 보기` expander를 열었을 때만 표시됩니다. 네이버/샘플 검색 결과에 전체 페이지 수가 없으면 0으로 표시되며, 사용자가 알고 있는 전체 페이지 수를 직접 입력한 뒤 책장에 추가할 수 있습니다. 입력한 페이지 수가 `books.csv`의 `total_pages`에 저장되고, 0이면 나중에 가족 책장에서 수정할 수 있습니다.


## 국립중앙도서관 API 선택 기능 / 단독 테스트

국립중앙도서관 API는 책장 추가 시 자동 호출하지 않습니다. 네트워크 환경에 따라 실패하거나 timeout이 발생할 수 있으므로, 필요할 때만 설정 화면의 개발자용 수동 테스트 또는 `test_nlk_api.py`로 응답을 확인합니다.

1. `.env`에 아래 값을 넣습니다.

```bash
NLK_CERT_KEY=발급받은_국립중앙도서관_인증키
```

2. `test_nlk_api.py` 상단의 `TEST_ISBN` 값을 확인하고, 필요하면 테스트할 ISBN으로 바꿉니다. 하이픈이 있어도 자동 정리됩니다.

```python
TEST_ISBN = "978-8954677158"
```

3. 아래 명령어로 실행합니다.

```bash
python test_nlk_api.py
```

스크립트는 요청 URL, 인증키가 마스킹된 요청 파라미터, `status_code`, 응답 텍스트 앞부분, 페이지 수 추출 결과를 출력합니다. Timeout 또는 RequestException 발생 시 긴 Traceback 대신 짧은 오류 메시지만 출력합니다. 페이지 수 추출에 실패하면 페이지 수가 들어 있을 가능성이 있는 후보 필드와 원본 응답 일부를 함께 보여줍니다.

앱 화면에서는 **설정 / 러너 관리** 화면의 **국립중앙도서관 페이지 수 조회 테스트** expander에서 수동 조회할 수 있습니다. 수동 조회도 5초 이내 timeout으로 제한되며, 인증키는 화면이나 로그에 노출하지 않습니다.

## 이번 버전의 책장 연결 방식

책 검색 결과에서 **책장에 추가**를 누르기 전에 읽을 러너을 선택합니다. 이 값은 `books.csv`의 `reader_member_id` 컬럼에 저장됩니다. 기존 CSV에 이 컬럼이 없어도 앱 실행 시 자동으로 컬럼을 추가하므로 기존 데이터가 바로 깨지지 않습니다.

같은 책을 같은 러너의 책장에 다시 추가하려고 하면 중복 등록하지 않고 안내 메시지를 표시합니다. 같은 책을 다른 러너가 읽는 경우에는 별도 책장 카드로 추가할 수 있습니다. 가족 책장 카드의 **전체 페이지 수 수정** expander에서 `total_pages`를 나중에 수정할 수 있고, 저장 후 책별 진행률이 다시 계산됩니다.

오늘의 독서 기록 화면에서는 먼저 러너를 선택하고, 그 러너의 `reader_member_id`와 일치하는 책만 책 드롭다운에 표시합니다. 예를 들어 엄마의 책장에만 추가한 책은 아빠를 선택했을 때 기록 입력 후보로 나오지 않습니다. 완독 감상이 저장된 책은 기본적으로 기록 후보에서 제외되며, **완독한 책도 보기**를 체크하면 다시 표시할 수 있습니다. 해당 러너의 책장에 등록된 책이 없거나 기록 가능한 읽는 중인 책이 없으면 안내 문구만 표시되고 저장 폼은 표시되지 않습니다.

책 검색 / 책 등록 화면은 검색 결과와 직접등록 폼이 섞여 보이지 않도록 등록 방식을 먼저 선택한 뒤 해당 화면만 렌더링합니다.

## 오늘의 독서 기록 / 기록 모아보기

기존의 **독서 기록 입력**, **좋았던 문장**, **한 줄 감상** 메뉴를 정리해 입력은 **오늘의 독서 기록**, 조회와 관리는 **기록 모아보기**로 통합했습니다.

오늘의 독서 기록 화면에서는 읽은 페이지 수 기록을 필수로 저장하고, 체크박스를 선택한 경우에만 좋았던 문장을 함께 저장합니다. 별점과 한 줄 감상은 **오늘 이 책을 완독했나요?**를 선택한 경우에만 입력·저장됩니다. 저장이 성공하면 책 선택은 **책을 선택하세요** 상태로 돌아가고, 페이지/메모/선택 입력값이 초기화됩니다. 러너 선택은 유지할 수 있지만 사용자가 다시 기록하려면 책을 명시적으로 다시 선택해야 하므로 같은 기록을 실수로 반복 저장할 위험을 줄입니다. 화면은 하나이지만 저장은 기존 CSV 구조를 유지합니다. 페이지 기록은 `reading_logs.csv`, 좋았던 문장은 `quotes.csv`, 완독 감상과 별점은 `reviews.csv`에 각각 저장됩니다.

기본 기록 방식은 **시작/끝 페이지로 계산**입니다. 시작 페이지와 끝 페이지를 입력하면 `end_page - start_page + 1`로 읽은 페이지 수가 자동 계산됩니다. 같은 러너가 같은 책을 이어 읽는 경우 기존 기록의 가장 큰 `end_page`를 확인해 다음 시작 페이지를 자동 제안합니다. 그림책이나 페이지 번호가 없는 책을 위해 **읽은 페이지 수 직접 입력** 방식도 지원합니다.


### 완독 감상과 별점

한 줄 감상은 일반 독서 기록마다 입력하지 않고, 책을 완독했을 때 남기는 구조입니다. **오늘 이 책을 완독했나요?**를 체크하면 별점, 한 줄 감상, 자세한 감상, 완독일 입력칸이 표시됩니다. 별점은 `⭐`부터 `⭐⭐⭐⭐⭐`까지 직관적으로 선택할 수 있으며, 내부적으로는 `reviews.csv`의 `rating` 컬럼에 1~5 숫자로 저장됩니다. 완독일 기본값은 오늘의 독서 기록 날짜입니다.

가족 책장에서는 해당 러너와 책 조합에 완독일이 있는 감상이 있으면 상태를 **완독**으로 표시합니다.

기록 모아보기 화면은 **독서 기록**, **좋았던 문장**, **한 줄 감상** 탭으로 구성되어 있습니다. 독서 기록 탭에서 각 기록별 **이 기록 삭제** 버튼을 누르면 바로 삭제하지 않고 “정말 이 독서 기록을 삭제할까요?” 확인 단계가 표시됩니다. **삭제 확인**을 한 번 더 눌렀을 때만 해당 `log_id`의 기록이 `reading_logs.csv`에서 삭제됩니다. 삭제 후 홈 대시보드, 가족 책장 진행률, 월간 리포트는 CSV를 다시 읽어 자동으로 재계산됩니다.

## 데이터 저장 방식

1차 MVP는 CSV 파일을 사용합니다.

- `books.csv`: 책 정보 및 책장에 연결된 읽는 사람
- `family_members.csv`: 러너 정보
- `reading_logs.csv`: 독서 기록
- `quotes.csv`: 좋았던 문장
- `reviews.csv`: 완독 감상, 별점 및 완독일 기록
- `settings.csv`: 독서마라톤 목표와 기간

나중에 필요하면 Supabase, Google Sheets, Firebase, SQLite 등으로 확장할 수 있습니다.

## 계산 규칙

- 개인 누적 페이지 = `pages_read` 합계
- 가중치 반영 페이지 = `pages_read × member_weight`
- 가족 전체 누적 페이지 = 모든 러너의 `weighted_pages` 합계
- 가족 진행률 = 가족 전체 누적 페이지 ÷ 가족 목표 페이지 × 100
- 책별 진행률 = 가능하면 해당 러너/책의 가장 큰 `end_page` ÷ `total_pages` × 100
- `end_page`가 없는 직접 입력 기록만 있는 경우에는 `pages_read` 합계 기준으로 fallback합니다.
- `total_pages`가 없거나 0이면 오류 없이 “전체 페이지 미입력” 상태로 표시합니다.

## 발표 시연 추천 흐름

1. 홈 화면의 **개발/시연용 도구** expander에서 샘플 데이터 생성 버튼을 누릅니다.
2. 대시보드에서 가족 전체 마라톤 트랙, 우리 가족 기여도 카드와 러너 카드를 보여줍니다.
3. 책 검색 / 책 등록 화면에서 네이버 API 또는 샘플 책을 검색합니다.
4. 읽을 러너을 선택한 뒤 책을 가족 책장에 추가합니다.
5. 오늘의 독서 기록 화면에서 읽은 구간을 입력하고, 필요하면 좋았던 문장을 함께 저장합니다. 책을 완독한 경우에만 별점과 한 줄 감상을 저장합니다. 저장 후 입력값이 초기화되는지 확인합니다.
6. 기록 모아보기에서 저장된 독서 기록, 좋았던 문장, 완독 감상을 확인합니다.
7. 필요하면 독서 기록을 삭제하고 재계산되는지 보여줍니다.
8. 홈 화면과 월간 리포트에서 진행률과 자동 요약문을 보여줍니다.

## v20 변경사항: 독서마라톤 선택 / 삭제 / 기본값 정리

이번 버전부터 보존된 독서마라톤 목록에서 과거 마라톤을 다시 active로 선택하거나, 테스트로 만든 마라톤을 삭제할 수 있습니다.

- `settings.csv`는 여러 개의 독서마라톤을 저장할 수 있도록 확장되었습니다.
- `books.csv`, `reading_logs.csv`, `quotes.csv`, `reviews.csv`에는 `marathon_id`가 추가됩니다.
- 기존 CSV에 `marathon_id`가 없으면 앱 실행 시 현재 active 마라톤으로 자동 보정됩니다.
- `설정 / 러너 관리` 화면에서 **새 독서마라톤 시작**을 사용할 수 있습니다.
- 새 마라톤을 시작해도 기존 러너는 유지되고, 기존 책장과 기록은 삭제되지 않습니다.
- 보존된 독서마라톤 목록에서 한 번에 하나의 마라톤만 **진행 중**으로 선택됩니다.
- 테스트로 만든 마라톤은 2단계 확인 후 삭제할 수 있습니다. 삭제 시 해당 마라톤의 책장, 독서 기록, 좋았던 문장, 완독 감상이 함께 삭제되지만 러너 정보는 유지됩니다.
- 마지막 남은 독서마라톤은 삭제할 수 없도록 막아두었습니다.
- 홈 대시보드와 오늘의 독서 기록은 active 마라톤 기준으로 표시됩니다.
- 가족 책장, 기록 모아보기, 월간 리포트에서는 마라톤을 선택해 과거 기록을 볼 수 있습니다.
- 새 마라톤 시작 기본값은 가족 월간 목표 2,000쪽, 단위는 페이지입니다.
- 샘플 데이터 생성 / 전체 초기화는 계속 개발/시연용 도구로만 사용하세요. 이 기능은 기존 데이터를 샘플 데이터로 덮어씁니다.
