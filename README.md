# 🏁 우리가족 독서마라톤

가족이 함께 읽은 책과 페이지 수, 좋았던 문장, 완독 감상을 기록하고  
가족 전체의 독서 목표 달성 과정을 마라톤처럼 시각화하는 Streamlit 웹앱입니다.

> 책을 많이 읽은 사람을 가리는 경쟁이 아니라,  
> 가족이 함께 하나의 결승선을 향해 달리는 **협동 독서마라톤**을 목표로 합니다.

## 1. 서비스 한 줄 소개

**우리가족 독서마라톤**은 네이버 책 검색 API로 책 정보를 불러오고, 가족 구성원이 읽은 페이지와 감상, 좋았던 문장을 기록하며, 가족 전체의 독서 목표 달성률을 시각적으로 확인할 수 있는 가족 독서 기록 웹앱입니다.

---

## 2. 프로젝트 개요 / 문제 정의

가족이 함께 독서 습관을 만들고 싶어도 실제로는 다음과 같은 어려움이 있습니다.

- 각자 어떤 책을 읽고 있는지 한눈에 보기 어렵습니다.
- 책을 어디까지 읽었는지 지속적으로 기록하기 어렵습니다.
- 아이들은 긴 독후감을 쓰기 부담스러워합니다.
- 가족 전체가 한 달 동안 얼마나 읽었는지 확인하기 어렵습니다.
- 독서가 개인 활동으로만 남고, 함께 응원하는 구조가 부족합니다.

이 프로젝트는 이러한 문제를 해결하기 위해 만들었습니다.

가족 구성원을 **러너**로 등록하고, 각자 읽는 책을 책장에 담은 뒤, 오늘 읽은 페이지와 좋았던 문장, 완독 감상을 기록합니다. 홈 대시보드에서는 가족 전체가 목표 페이지까지 얼마나 왔는지 마라톤 트랙 형태로 보여줍니다.

---

## 3. 핵심 콘셉트: 협동 독서마라톤

이 앱은 가족 구성원끼리 순위를 경쟁하는 서비스가 아닙니다.

가족 전체가 하나의 목표를 향해 함께 달리는 구조입니다.

- 가족 목표 페이지를 설정합니다.
- 각 러너가 읽은 페이지가 가족 전체 독서 거리로 누적됩니다.
- 어린이와 유아도 독서 난이도에 따라 가중치를 반영할 수 있습니다.
- 한 달 독서마라톤이 끝나면 기록은 보존하고 새 마라톤을 시작할 수 있습니다.

## 4. 주요 기능 요약

### 홈 / 대시보드

- 가족 전체 독서마라톤 진행률 표시
- 목표 페이지, 누적 페이지, 남은 거리 표시
- 마라톤 트랙형 진행률 UI
- 러너별 협동 기여도 표시
- 규칙 기반 응원 문구 표시
- 최근 독서 기록 피드 표시

### 책 검색 / 책 등록

- 네이버 책 검색 API 연동
- 책 제목 검색
- ISBN 검색
- 검색어 입력 후 Enter 검색 지원
- 검색 결과 50개 조회 후 10개 단위 페이지네이션
- 책 소개는 expander로 접어서 표시
- 책장 추가 시 러너 선택
- 전체 페이지 수 직접 입력 가능
- API 실패 시 샘플 검색으로 fallback

### 가족 책장

- 러너별 책장 카드 표시
- 책 표지, 제목, 저자, 출판사, ISBN 표시
- 읽는 사람 표시
- 읽는 중 / 완독 상태 표시
- 책별 진행률 표시
- 전체 페이지 수 수정 가능
- 기록이 없는 책은 책장에서 제거 가능
- 기록이 없는 책은 읽는 러너 변경 가능
- 독서 기록, 좋았던 문장, 완독 감상 등 연결된 기록이 있는 책은 데이터 보호를 위해 제거/러너 변경 제한

### 오늘의 독서 기록

- 러너 선택
- 해당 러너의 현재 마라톤 책장에 있는 책만 선택
- 시작 페이지 / 끝 페이지 입력 시 읽은 페이지 수 자동 계산
- 책 전체 페이지 수를 넘는 기록 입력 방지
- 직접 입력 방식 지원
- 이전 기록 기준 다음 시작 페이지 자동 제안
- 좋았던 문장 선택 입력
- 완독 시 별점과 한 줄 감상 저장
- 저장 후 입력 폼 초기화

### 기록 모아보기

- 독서 기록 목록 조회
- 좋았던 문장 목록 조회
- 한 줄 감상 및 별점 조회
- 독서 기록 삭제 전 확인 후 삭제
- 좋았던 문장 삭제 전 확인 후 삭제
- 한 줄 감상 및 완독 감상 삭제 전 확인 후 삭제
- 마라톤별 기록 조회

### 월간 리포트

- 선택한 독서마라톤 기준 리포트 생성
- 총 읽은 페이지
- 목표 달성률
- 러너별 누적 페이지
- 가장 많이 기여한 러너
- 가장 꾸준히 기록한 러너
- 완독한 책
- 이달의 문장
- 규칙 기반 자동 요약문

### 설정 / 러너 관리

- 러너 추가
- 러너 정보 수정
- 러너 삭제
- 러너 유형 설정
  - 성인
  - 청소년
  - 어린이
  - 유아
- 러너별 가중치 설정
- 새 독서마라톤 시작
- 보존된 독서마라톤 선택 / 삭제
- 샘플 데이터 생성 및 전체 초기화
- 관리자 모드로 쓰기/삭제/초기화 기능 보호

## 17. API 사용 방식

### 네이버 책 검색 API

네이버 책 검색 API를 사용하면 책 제목 또는 ISBN으로 책 정보를 검색할 수 있습니다.

가져오는 정보는 다음과 같습니다.

- 책 제목
- 저자
- 출판사
- ISBN
- 표지 이미지
- 책 소개
- 출간일

로컬 실행 시 `.env` 파일에 아래 값을 입력합니다.

```env
NAVER_CLIENT_ID=발급받은_CLIENT_ID
NAVER_CLIENT_SECRET=발급받은_CLIENT_SECRET
```

Streamlit Community Cloud에서는 Secrets에 같은 값을 등록합니다.

```toml
NAVER_CLIENT_ID = "발급받은_CLIENT_ID"
NAVER_CLIENT_SECRET = "발급받은_CLIENT_SECRET"
```

## 관리자 모드 설정

공개 Streamlit 앱에서는 누구나 접속할 수 있으므로, 책 추가·기록 저장·삭제·초기화 같은 변경 기능은 관리자 모드에서만 사용할 수 있게 보호합니다.

관리자 모드가 아닐 때도 아래 화면은 읽기 전용으로 확인할 수 있습니다.

- 홈 / 대시보드
- 가족 책장
- 기록 모아보기
- 월간 리포트

관리자 모드가 아닐 때 제한되는 기능은 다음과 같습니다.

- 책장에 책 추가
- 오늘의 독서 기록 저장
- 기록 삭제
- 가족 책장 관리
- 러너 추가 / 수정 / 삭제
- 새 독서마라톤 시작 / 삭제
- 샘플 데이터 생성 / 전체 초기화

### Streamlit Secrets 설정

Streamlit Community Cloud의 앱 설정에서 **Secrets**에 아래 값을 추가합니다.

```toml
ADMIN_PASSWORD = "원하는_관리자_비밀번호"
NAVER_CLIENT_ID = "발급받은_CLIENT_ID"
NAVER_CLIENT_SECRET = "발급받은_CLIENT_SECRET"
```

앱 실행 후 사이드바의 **관리자 모드** 입력칸에 비밀번호를 입력하면 쓰기 기능을 사용할 수 있습니다.

로컬 개발 편의를 위해 `ADMIN_PASSWORD`가 설정되어 있지 않은 경우에는 관리자 권한을 허용하되, 공개 배포 시에는 반드시 Secrets에 `ADMIN_PASSWORD`를 설정하는 것을 권장합니다.

### API 키가 없어도 작동

API 키가 없거나 API 호출에 실패해도 앱이 중단되지 않습니다.

이 경우 샘플 도서 데이터 기반으로 검색과 등록 기능을 사용할 수 있습니다. 따라서 심사나 발표 환경에서도 안정적으로 시연할 수 있습니다.

### 국립중앙도서관 ISBN API

국립중앙도서관 ISBN API는 선택 기능으로 분리했습니다.

현재 앱에서는 책장 추가 시 자동 호출하지 않습니다. 네트워크 환경에 따라 응답 지연 또는 timeout이 발생할 수 있어, MVP 핵심 흐름을 방해하지 않도록 개발자용 수동 테스트 기능만 제공합니다.

## 17. 추천 데모 시나리오

심사 또는 발표 시 아래 순서로 시연할 수 있습니다.

### Step 1. 홈 대시보드 확인

- 가족 독서마라톤 목표 확인
- 가족 전체 진행률 확인
- 마라톤 트랙 UI 확인
- 러너별 협동 기여도 확인

### Step 2. 책 검색 및 책장 추가

- 책 검색 / 책 등록 메뉴로 이동
- 책 제목 또는 ISBN 입력
- 네이버 책 검색 결과 확인
- 책 소개 expander 확인
- 전체 페이지 수 입력
- 읽을 러너 선택 후 책장에 추가

### Step 3. 가족 책장 확인

- 가족 책장 메뉴로 이동
- 방금 추가한 책이 표시되는지 확인
- 읽는 러너와 진행률 확인
- 기록이 없는 책은 책장 관리에서 제거하거나 읽는 러너를 변경할 수 있음
- 기록이 있는 책은 데이터 보호를 위해 제거/러너 변경이 제한됨

### Step 4. 오늘의 독서 기록 입력

- 러너 선택
- 책 선택
- 시작 페이지와 끝 페이지 입력
- 읽은 페이지 수 자동 계산 확인
- 끝 페이지가 책 전체 페이지 수를 넘으면 저장되지 않는지 확인
- 좋았던 문장 선택 입력
- 완독 시 별점과 한 줄 감상 저장

### Step 5. 기록 모아보기

- 독서 기록 조회
- 좋았던 문장 조회
- 한 줄 감상과 별점 조회
- 독서 기록, 좋았던 문장, 한 줄 감상 삭제 전 확인 기능 확인

### Step 6. 월간 리포트

- 선택한 독서마라톤 기준 리포트 확인
- 총 페이지, 목표 달성률, 완독한 책, 이달의 문장 확인
- 규칙 기반 자동 요약문 확인

### Step 7. 새 독서마라톤 시작

- 설정 / 러너 관리 메뉴로 이동
- 새 독서마라톤 시작
- 기존 기록은 보존되고 새 마라톤이 active 상태가 되는지 확인
- 보존된 독서마라톤 목록에서 이전 마라톤을 다시 선택할 수 있는지 확인

## 17. MVP 구현 범위

이번 버전은 가족 사용을 위한 1차 MVP입니다.

포함된 기능:

- 책 검색
- 책장 등록
- 가족 러너 관리
- 독서 기록
- 좋았던 문장
- 완독 감상
- 별점
- 월간 리포트
- 새 독서마라톤 시작
- 보존된 마라톤 선택 / 삭제
- 샘플 데이터 기반 시연

제외한 기능:

- 로그인
- 회원가입
- 결제
- SNS 공유
- OCR
- 바코드 스캔
- 모바일 앱
- 실제 AI API 연동
- 복잡한 권한 관리

## 17. 향후 확장 방향

### 데이터 저장 확장

현재는 CSV 기반으로 저장하지만, 향후 다음 방식으로 확장할 수 있습니다.

- SQLite
- Google Sheets
- Supabase
- Firebase

### 게임화 요소 확장

독서마라톤 콘셉트를 더 강화하기 위해 다음 기능을 추가할 수 있습니다.

- 배지 시스템
- 가족 협동 미션
- 이벤트 기간 보너스
- 아이템 획득
- 완주 인증서
- 월간 독서 리포트 PDF 생성

### 책 API 확장

현재는 네이버 책 검색 API를 중심으로 사용합니다.

향후 다음 API를 추가할 수 있습니다.

- 국립중앙도서관 ISBN 서지정보 API
- 알라딘 Open API
- 베스트셀러 / 신간 추천 API

### AI 기능 확장

현재 MVP에서는 실제 AI API를 사용하지 않습니다.

향후에는 다음 기능을 검토할 수 있습니다.

- 가족 독서 리포트 자동 생성
- 아이 수준에 맞춘 독서 질문 추천
- 좋았던 문장 기반 대화 주제 추천
- 완독 감상 요약

## 17. 프로젝트의 차별점

이 프로젝트는 단순한 독서 기록장이 아니라, 가족이 함께 쓰는 협동형 독서마라톤입니다.

- 책을 검색해 쉽게 등록할 수 있습니다.
- 아이도 짧게 기록할 수 있습니다.
- 가족 전체 목표 달성률이 한눈에 보입니다.
- 기존 기록을 보존한 채 새 독서마라톤을 시작할 수 있습니다.
- API 키가 없어도 샘플 데이터로 안정적인 시연이 가능합니다.
- 복잡한 상용 서비스가 아니라, 실제 가족이 바로 써볼 수 있는 작고 작동하는 MVP입니다.

---

## 17. 개발 목적

이 프로젝트의 목표는 거대한 플랫폼을 만드는 것이 아니라, 가족이 실제 생활에서 사용할 수 있는 작은 웹앱을 빠르게 만들고, 기능이 실제로 작동하는 MVP로 구현하는 것입니다.

> “우리 가족이 책을 더 즐겁게 읽고, 서로의 독서 여정을 확인할 수 있는가?”

이 질문을 기준으로 기능을 설계하고 구현했습니다.
