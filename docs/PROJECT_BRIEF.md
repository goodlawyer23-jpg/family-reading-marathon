\# PROJECT\_BRIEF.md



\## 1. 프로젝트명



우리가족 독서마라톤



\## 2. 한 줄 소개



우리가족 독서마라톤은 책 검색 API로 책 정보를 자동으로 불러오고, 가족 구성원이 읽은 페이지와 감상, 좋았던 문장을 함께 기록하며 가족 전체의 독서 완주 과정을 시각화하는 웹페이지다.



\## 3. 프로젝트 목적



이 프로젝트의 목적은 가족이 함께 독서 습관을 만들고, 각자 읽는 책과 독서 진행 상황을 한눈에 확인할 수 있는 작은 웹페이지를 만드는 것이다.



단순히 “몇 권 읽었는지”를 세는 것이 아니라, 가족이 함께 하나의 독서마라톤을 완주하는 경험을 만드는 것이 핵심이다.



\## 4. 문제정의



가족이 함께 책을 읽으려 해도 다음과 같은 문제가 있다.



\* 각자 무슨 책을 읽고 있는지 잘 공유되지 않는다.

\* 책을 어디까지 읽었는지 한눈에 보기 어렵다.

\* 독서 기록을 남기려면 책 제목, 저자, 출판사, 표지 등을 직접 입력해야 해서 번거롭다.

\* 아이들은 글로 감상을 길게 남기기 어렵다.

\* 한 달 동안 가족 전체가 얼마나 읽었는지 확인하기 어렵다.

\* 독서가 개인 활동으로만 남고, 가족이 함께 응원하는 구조가 부족하다.



\## 5. 해결 아이디어



우리가족 독서마라톤은 책 검색 API를 활용해 책 정보를 자동으로 불러오고, 가족 구성원이 읽은 페이지와 짧은 감상, 좋았던 문장을 쉽게 기록할 수 있게 한다.



기록된 데이터는 가족 전체 목표 대비 진행률, 구성원별 누적 페이지, 최근 기록, 월간 리포트로 시각화된다.



즉, 책을 “등록하는 번거로움”은 줄이고, 가족이 함께 “읽고 확인하고 응원하는 경험”을 강화한다.



\## 6. 핵심 사용자



\### 1. 가족 구성원



\* 부모

\* 초등학생 자녀

\* 아직 긴 글쓰기가 어려운 어린이



\### 2. 독서 습관을 만들고 싶은 가정



가족이 함께 책을 읽고 싶지만 꾸준한 기록과 공유가 어려운 가정.



\### 3. 경진대회 심사자



생활 속 문제를 웹서비스로 어떻게 해결했는지 확인하는 사용자.



\## 7. 핵심 사용 시나리오



\### 시나리오 1. 책 등록



사용자가 책 제목을 검색한다.

네이버 책 검색 API 등을 통해 책 표지, 저자, 출판사, ISBN, 책 소개가 표시된다.

사용자는 검색 결과 중 하나를 선택해 가족 책장에 추가한다.



\### 시나리오 2. 독서 기록



가족 구성원 중 한 명을 선택한다.

오늘 읽은 책과 페이지 수를 입력한다.

필요하면 시작 페이지, 끝 페이지, 짧은 메모를 남긴다.

기록을 저장하면 개인 누적 페이지와 가족 전체 진행률이 자동으로 업데이트된다.



\### 시나리오 3. 가족 대시보드



홈 화면에서 이번 달 가족 목표 페이지, 현재 누적 페이지, 진행률, 구성원별 누적 페이지, 최근 기록을 확인한다.

가족 전체가 목표를 향해 얼마나 왔는지 직관적으로 볼 수 있다.



\### 시나리오 4. 좋았던 문장 기록



책을 읽다가 기억하고 싶은 문장이 있으면 책, 페이지 번호, 문장, 내 생각을 남긴다.

가족은 각자가 어떤 문장에 반응했는지 볼 수 있다.



\### 시나리오 5. 월간 리포트



한 달이 끝나면 가족 전체가 읽은 페이지, 완독한 책, 가장 많이 읽은 사람, 가장 꾸준히 기록한 사람, 이달의 문장을 확인한다.



\## 8. API 활용 방향



\### 네이버 책 검색 API



MVP의 1순위 API다.

책 제목 검색, 표지, 저자, 출판사, ISBN, 책 소개, 출간일, 외부 링크를 가져오는 데 활용한다.



\### 국립중앙도서관 ISBN 서지정보 API



책의 전체 페이지 수를 보강하는 데 활용한다.

ISBN 기반으로 페이지 수를 가져올 수 있으면 책별 진행률 계산에 도움이 된다.



\### 알라딘 Open API



확장 기능에 활용한다.

베스트셀러, 신간, 편집자 추천, 카테고리 정보 등을 활용해 “읽고 싶은 책” 후보를 제안할 수 있다.



\## 9. 핵심 가치



\### 1. 기록 부담 감소



책 정보를 API로 불러와 사용자가 직접 입력해야 하는 정보를 줄인다.



\### 2. 가족 공유



각자 읽는 책과 진행 상황을 가족 전체가 확인할 수 있다.



\### 3. 시각적 동기부여



누적 페이지, 진행률, 트랙, 배지 등을 통해 독서 과정을 눈에 보이게 만든다.



\### 4. 아이 친화성



긴 감상문 대신 읽은 페이지, 이모지, 한 줄 감상, 좋았던 문장 중심으로 기록할 수 있다.



\### 5. 실제 사용 가능성



경진대회용 데모에 그치지 않고, 우리 가족이 실제로 7월 독서마라톤에 사용할 수 있는 웹페이지를 지향한다.



\## 10. 발표용 핵심 메시지



“우리가족 독서마라톤은 가족이 함께 읽는 책과 독서 진행 상황을 한곳에서 확인할 수 있게 해주는 웹페이지입니다. 책 API를 활용해 책 정보를 자동으로 불러오고, 가족 구성원이 읽은 페이지와 감상, 좋았던 문장을 기록하면 가족 전체의 독서 목표 달성률이 시각적으로 표시됩니다. 독서량 경쟁보다는 함께 읽고, 서로의 페이스를 확인하고, 좋은 문장을 나누는 경험을 목표로 합니다.”



\## 11. 데모 시나리오



1\. 홈 화면에서 7월 가족 독서마라톤 목표를 확인한다.

2\. 책 검색 화면에서 책 제목을 입력한다.

3\. API 검색 결과에서 책을 선택해 가족 책장에 추가한다.

4\. 가족 구성원을 선택하고 오늘 읽은 페이지를 기록한다.

5\. 대시보드에서 가족 전체 진행률과 개인별 누적 페이지가 업데이트되는 것을 확인한다.

6\. 좋았던 문장을 기록한다.

7\. 월간 리포트에서 가족 전체 독서 결과를 확인한다.



\## 12. 최종 지향점



이 프로젝트는 거대한 플랫폼이 아니라, 작지만 실제 생활에 쓸 수 있는 가족 독서 웹페이지를 목표로 한다.



\* 우리 가족이 실제로 쓸 수 있어야 한다.

\* 아이들도 자기 기록을 볼 수 있어야 한다.

\* 책 표지와 진행률이 예쁘게 보여야 한다.

\* 기록을 남기는 과정이 부담스럽지 않아야 한다.

\* 한 달 뒤 가족 독서의 흔적이 남아야 한다.

\* 경진대회에서는 “생활 속 문제를 작은 웹서비스로 해결했다”는 점이 드러나야 한다.
